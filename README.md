
**FinUp** — это система мониторинга и анализа финансовых данных пользователя с помощью AI.
Главная цель проекта — помочь пользователю следить за своими расходами, доходами и получать рекомендации на основе
анализа его финансов.
То есть проект объединяет три направления:
Сбор данных о расходах и доходах (BudgetService, TransactionService, UserService).
Анализ этих данных и формирование отчетов (ежедневные и еженедельные отчеты).
Помощь с рекомендациями и советами с помощью AI (AiService).
2. Структура проекта и её смысл
   **Что делает?	Зачем нужен?**
3.   UserService	Управляет пользователями, их профилями и ID	Чтобы можно было идентифицировать пользователей и
     связывать их с транзакциями и отчетами
4. TransactionService	Хранит транзакции, создаёт отчёты по ним (день/неделя)	Служит источником данных для отчетов и
   анализа расходов/доходов
5. BudgetService	Составляет ежедневные и еженедельные отчеты	Чтобы пользователь видел статистику и мог принимать
   финансовые решения
6.   AiService	Классифицирует текст и даёт советы	Чтобы автоматизировать рекомендации пользователю
     (например, куда экономить)
7.  Gateway + Eureka	Маршрутизация запросов между сервисами	Чтобы все микросервисы могли общаться друг с другом,
    а фронтенд видел единый API
  
*****Откуда берутся данные*****
    Пользователь создаёт транзакцию через /users/user/spend.
    TransactionService сохраняет её в базу данных.
    BudgetService берёт все транзакции пользователя и формирует отчеты:
    daily — ежедневные расходы/доходы
    weekly — суммарные показатели за неделю
    AiService анализирует текстовые данные пользователя и создаёт советы:
    Можно присылать текст о тратах или финансовых целях
    AI возвращает категорию или совет, куда лучше потратить/сэкономить

# Документация по REST API

## 1. AiService

**Базовый URL:** `/ai`

### Эндпоинты

#### 1.1. Классификация текста

* **URL:** `/ai/category`
* **Метод:** `POST`
* **Параметры:**

    * `text` (String) — текст для классификации
* **Описание:** Отправляет текст в AI-сервис для определения категории.
* **Ответ:** `String` — название категории.

#### 1.2. Получение советов

* **URL:** `/ai/advice`
* **Метод:** `POST`
* **Параметры:**

    * `text` (String) — текст для анализа и получения совета
* **Описание:** Возвращает совет от AI на основе текста.
* **Ответ:** `String` — рекомендация/совет.

---

## 2. BudgetService

**Базовый URL:** `/api/budget`

### Эндпоинты

#### 2.1. Отчеты Weekly

* **GET** `/report/weekly` — получить все еженедельные отчеты
* **GET** `/report/weekly/{reportId}` — получить отчет по `reportId`
* **GET** `/report/weekly/user/{userId}` — получить все отчеты пользователя по `userId`
* **GET** `/report/weekly/last/user/{userId}` — получить последний отчет пользователя

#### 2.2. Отчеты Daily

* **GET** `/report/daily` — получить все ежедневные отчеты
* **GET** `/report/daily/{reportId}` — получить отчет по `reportId`
* **GET** `/report/daily/user/{userId}` — получить все отчеты пользователя по `userId`
* **GET** `/report/daily/last/user/{userId}` — получить последний отчет пользователя

**Примечание:** Все эндпоинты возвращают соответствующие DTO объекты `ReportDaily` или `ReportWeekly`.

---

## 3. TransactionService

**Базовый URL:** `/transactions`

### Эндпоинты

* **GET** `/hello` — проверка сервиса (возвращает `"hello"`)
* **GET** `/transactions` — получить список всех транзакций
* **GET** `/userTransactions/{id}` — получить все транзакции пользователя по `id`
* **GET** `/reportWeekly/{id}` — генерация еженедельного отчета пользователя по `id` (возвращает `Map<String, Double>`)
* **GET** `/reportDaily/{id}` — генерация ежедневного отчета пользователя по `id`
* **GET** `/ids` — получить список всех id пользователей

---

## 4. UserService

**Базовый URL:** `/users`

### Эндпоинты

#### 4.1. Пользователи

* **GET** `/user` — получить профиль пользователя по токену авторизации
* **GET** `/user/{id}` — получить профиль пользователя по `id`
* **GET** `/` — получить список всех пользователей
* **GET** `/id` — получить список всех ID пользователей

#### 4.2. Обновление профиля

* **PUT** `/user/update` — обновить профиль пользователя

    * **Заголовок:** `Authorization`
    * **Тело запроса:** JSON с полями пользователя

#### 4.3. Транзакции

* **POST** `/user/spend` — создать новую транзакцию

    * **Заголовок:** `Authorization`
    * **Тело запроса:** `TransactionDTO`

#### 4.4. Отчеты

* **GET** `/user/report/weekly` — получить все еженедельные отчеты пользователя
* **GET** `/user/report/weekly/last` — получить последний еженедельный отчет
* **GET** `/user/advice/weekly` — получить советы по недельным отчетам
* **GET** `/user/report/daily/last` — получить последний ежедневный отчет
* **GET** `/user/advice/daily` — получить советы по ежедневным отчетам
